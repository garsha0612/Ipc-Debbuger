IPC Mechanism Reference
⇄ PIPE
Half-duplex byte stream between related processes. Low overhead, kernel-buffered. Best for sequential producer-consumer patterns.
Latency~0.05ms
Buffer64KB kernel
DirectionUnidirectional
Syscallpipe(2) / read/write
≡ MESSAGE QUEUE
Discrete message passing with priority support. Decoupled sender/receiver. Persistent until consumed. POSIX mq_send/mq_receive.
Latency~0.12ms
PriorityYes (0–31)
PersistenceUntil unlink
Syscallmq_open / mq_send
◈ SHARED MEMORY
Zero-copy memory region shared across processes. Fastest IPC mechanism. Requires synchronization via semaphores or mutexes to prevent races.
Latency~0.01ms
CopyZero-copy
SyncMutex/Semaphore
Syscallshm_open / mmap
